Домашнее задание. Аутентификация

Задание выполняется индивидуально.

Цель задания

Научиться реализовывать аутентификацию между несколькими микросервисами.

Условия

В рамках этого задания вам необходимо создать два микросервиса, один из которых будет аутентифицировать пользователя и выдавать ему JWT-токен, а второй будет принимать от пользователя сообщение и класть его в базу данных от его имени, проверяя перед этим корректность этого JWT-токена.

Для этого выполните следующие шаги:

Создайте приложение для аутентификации, которое будет регистрировать и аутентифицировать пользователей и соответствовать следующим условиям:
Приложение должно будет работать с одной таблицей базы данных users, имеющей следующую структуру: id, email, password. В поле password должны храниться надежно захэшированные пароли пользователей.
Приложение должно принимать POST-запрос на регистрацию пользователя, в котором в JSON-формате должны содержаться email и пароль. В ответ на этот запрос приложение должно возвращать ошибку, если email неверный, уже существует или пароль небезопасный. Если все перечисленное корректно, приложение должно возвращать пустой ответ с кодом 201.
Приложение должно также принимать POST-запрос аутентификации с email и паролем в формате JSON и проверять их корректность. Если некорректен email или некорректен пароль, должен возвращаться пустой ответ с кодом 401. Если корректны, то ответ с кодом 200 и JWT-токеном, содержащим user_id и срок действия.
Создайте приложение для публикации постов, которое в POST-запросе будет:
Принимать текст сообщения в виде JSON-объекта с ключом message и JWT-токеном в заголовке:
Authorization: Bearer <token>
Проверять подпись переданного токена и срок его действия.
Если все корректно, то сохранять сообщение в таблицу messages, содержащую поля id, user_id, time, message. В поле time должны быть дата и время сохранения сообщения. В ответ возвращать код 201 и пустой ответ.
Если подпись токена некорректна, возвращать пустой ответ с кодом 400.
Если срок действия токена истек, возвращать пустой ответ с кодом 401.
Пропишите для обоих приложений Dockerfile. Соберите и запустите контейнеры и приложения при помощи docker compose. Базу данных поместите в третий контейнер.
Создайте в Postman три запроса — на регистрацию пользователя, аутентификацию и создание сообщения с отправкой JWT-токена. Проверьте, что все работает корректно. Исправьте ошибки, если они будут.

## Реализация

В проекте реализованы два микросервиса:

- `auth_service` на порту `8000`: регистрация и аутентификация пользователей.
- `posts_service` на порту `8001`: создание сообщений по валидному JWT.

PostgreSQL запускается отдельным контейнером и содержит таблицы `users` и `messages`.

## Запуск

```bash
docker compose up --build
```

## Запросы для Postman

### Регистрация

`POST http://localhost:8000/register`

```json
{
  "email": "user@example.com",
  "password": "StrongPass1"
}
```

При успешной регистрации возвращается пустой ответ с кодом `201`.

### Аутентификация

`POST http://localhost:8000/login`

```json
{
  "email": "user@example.com",
  "password": "StrongPass1"
}
```

При успешной аутентификации возвращается код `200` и JWT:

```json
{
  "token": "<jwt>"
}
```

### Создание сообщения

`POST http://localhost:8001/messages`

Заголовок:

```text
Authorization: Bearer <jwt>
```

Тело:

```json
{
  "message": "Hello from authenticated user"
}
```

При успешном сохранении сообщения возвращается пустой ответ с кодом `201`.
